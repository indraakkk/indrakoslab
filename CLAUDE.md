## Overview

Personal website (indrakoslab) built with TanStack Start (React 19) + Vite + Tailwind CSS v4, deployed to Vercel via the Nitro Vite plugin. Package manager is pnpm (Node >= 20). There are no tests and no linter configured; use `npx tsc --noEmit` to typecheck.

## Commands

- `pnpm dev` — dev server on port 3000
- `pnpm build` — production build (Vite + Nitro, Vercel nodejs20.x target)
- `pnpm start` — serve the built output (`.output/server/index.mjs`)

Note: the README is outdated (it mentions Turborepo/Vue/Nuxt); the actual stack is TanStack Start + React.

## Architecture

- **Routing**: File-based routes in `src/routes/` via TanStack Router. `src/routeTree.gen.ts` is generated by the TanStack Start Vite plugin during dev/build — never edit it by hand. `src/router.tsx` exports `getRouter()` consumed by TanStack Start.
- **Root shell**: `src/routes/__root.tsx` defines the full HTML document (`shellComponent`), head/meta/stylesheet links, and the shared layout (Navbar, footer). Global CSS lives in `src/styles/globals.css` and is linked via the `?url` import.
- **Theming (SSR cookie-based)**: `src/lib/theme.ts` defines TanStack server functions (`createServerFn`) that read/write a `theme` cookie, validated with Zod. The root route's loader fetches the theme so SSR renders the correct `class` on `<html>` (no flash). `src/components/theme-provider.tsx` exposes `useTheme()`; setting the theme calls the server fn then `router.invalidate()` to re-run the loader.
- **Monaco editor**: `src/components/editor-monaco.tsx` lazily imports `monaco-editor-core`, `shiki`, and `@shikijs/monaco` inside `useEffect` (client-only, keeps them out of the SSR bundle). Themes are `tokyo-night` (dark) / `one-light` (light), synced with the theme provider.
- **Path alias**: `@/*` → `./src/*` (tsconfig paths + vite-tsconfig-paths).
- **UI**: shadcn-style components in `src/components/ui/` using class-variance-authority; `cn()` helper in `src/lib/utils.ts` (clsx + tailwind-merge). Tailwind v4 (CSS-first config — no tailwind.config file; tokens live in `globals.css`).
